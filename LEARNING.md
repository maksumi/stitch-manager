# LEARNING

## Propósito

Registrar conocimientos que el desarrollador necesite aprender o reforzar para avanzar en el proyecto.

---

## Convenciones

| Estado | Significado |
|--------|-------------|
| Pendiente | Tema identificado como necesario |
| En estudio | El desarrollador está aprendiendo el tema |
| Validación | Verificando la comprensión del tema |
| Comprendido | Suficiente entendimiento para aplicarlo |
| Aplicado | Conocimiento utilizado dentro del proyecto |

---

## Registro

| Tema | Estado | Nota |
|------|--------|------|
| Modelado de dominio | Aplicado | Definimos objetos, relaciones, cardinalidades y reglas de negocio de Stitch Manager de principio a fin |
| Diseño de APIs | Pendiente | Se tocará al definir los endpoints de la primera rebanada |
| Modelado de datos / ORM | Pendiente | Se tocará al definir el schema de Prisma |
| Autenticación básica | Pendiente | Se tocará al implementar login de operador |
